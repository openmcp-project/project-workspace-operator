# v1 Support

While this operator has been refactored into a platform service, meaning it can only be deployed on v2 openmcp landscapes, there is some support for hybrid landscapes (serving the v1 and the v2 resources in parallel) available. 

> [!WARNING]
> v1 is deprecated and this feature will be removed in the foreseeable future.

To activate the v1 support mode, this platform service's `run` command needs to be executed with an additional `--v1` argument. Note that this requires a non-standard `PlatformService` resource which adds this argument to the `run` command. The `task platformservice-v1` command can be used to generate a fitting manifest.

The effect of v1 support mode is simply that `ManagedControlPlane` and `ClusterAdmin` are added to the list of resources that workspace members have permissions for (read only or read and write, depending on role) and to the list of resources which block the deletion of a workspace, if they exist within that workspace's namespace.

> [!NOTE]
> The same effects as when enabling v1 support mode can also be achieved by adding the aforementioned resources to the corresponding fields in the [configuration](config.md) instead.
